১) Scope ও সত্য-চেক নীতি

এই অডিটের নিয়ম: “যা কোডে নেই, তা দাবি করা হবে না”। Single Source of Truth হিসেবে ধরা হয়েছে: (১) API route code, (২) mini-services/stock-server WS implementation, (৩) Prisma DB writes/reads, (৪) MCP execution logs.

২) আর্কিটেকচার: WS → API → DB (Single Source of Truth)

বর্তমান ডিজাইনে “truth” একটি জায়গায়: Next.js API + DB। Stock WS server সরাসরি provider কে কল করে না—এটা /api/chat/stream কল করে (SSE), এবং /api/stock/events দিয়ে DB-তে event ingest করে।

৩) Session persistence (48h) + editor connection audit

UI dashboard (docs/index.html minimal) LocalStorage-এ sessionId TTL=48h ধরে রাখে। WS server session.resume এলে DB-backed API দিয়ে validate করে—এটাই “real session” gate। 

Truth points
- WS server: session.resume → sessionExists(sessionId) (HTTP GET /api/sessions/{id})
- If ok: client.hasRealSession = true; emits event: session.resumed
- DB ingest: /api/stock/events type=bind_chat_session, client_ping
- Audit API: GET /api/editor/connections (48h window)
  • Strength: sessionId fake হলে WS server accept করে না (API validation দরকার)।
  • Risk: dashboard যদি file:// থেকে খোলে, fetch/WS CORS ব্লক হবে (এটা UI তে warning যোগ করা হয়েছে)।
Session integrity hardening (প্রস্তাব)

এখনকার integrity “exists check” এ নির্ভর। আরো শক্ত করতে চাইলে:

  • Session binding token: /api/sessions create করলে server-signed token ফেরত দিবে; WS resume এ token validate হবে।
  • Client identity fingerprint: clientId + userAgent hash + ip constraints (soft) লগে রাখলে hijack detection সহজ হবে।
  • Rate limiting: /api/stock/events এ per-client throttling already আছে (ping 15s), কিন্তু ws_request_log এ rate guard দরকার হতে পারে।

Mapping-based compact prompt (প্রস্তাবিত)

লক্ষ্য: prompt ছোট হবে, কিন্তু intent/persona হারাবে না। পদ্ধতি: systemPrompt কে “human prose” না রেখে “compact protocol header” বানাও—যেটা config থেকে map হয়। 

Compact format idea (example)

[ZC]
ID=ZombieCoder; OWNER=Sahon Srabon; ORG=Developer Zone
PERS=TotkaSahon (if persona)
LANG=bn; TECH=ts
RULES=NO_HALLUCINATION|NO_REPEAT|IDENTITY_ANCHOR|BENGALI_DEFAULT
CAP=code|audit|ws|db

[AGENT]
NAME=...; TYPE=coder; PROVIDER_ID=...; MODEL=...

[MEM]
A: topic1=...; topic2=...
C: type1=...; type2=...

Why it preserves intent:
- Each key maps to stable semantics
- Model sees constraints as explicit flags, not verbose narrative
  • Do: behavioral rules ছোট করে symbolic flags + ১ লাইনের explanation রাখো।
  • Do: memory entries truncate + hash anchor (topicId) রাখো যাতে recall possible।
  • Avoid: একই identity prompt বারবার long form দেয়া।

Providers (OpenAI/Ollama/Gemini/llama.cpp) implement করে: chat() এবং chatStream()। কিন্তু tool calling orchestration (যেমন OpenAI tool_calls) নেই। তাই “agent persona অনুযায়ী tools ব্যবহার করতে পারছে কিনা” প্রশ্নে—truth হলো:

  • Tools exist: MCP tool execution real।
  • Model auto-tool-use missing: LLM response parse করে tool invoke করে আবার model-এ feed করার controller নেই।

প্রস্তাব: “Tool Controller” layer 

Minimal Tool Controller (design)
1) Chat request arrives
2) Build systemPrompt (compact)
3) Ask provider for response
4) Parse response for a strict tool directive format (not model-native, your own protocol)
   e.g. "[TOOL] name=shell_execute args={...}"
5) Execute via POST /api/mcp/execute (agentId+sessionId included)
6) Append tool result as a tool message (role=tool) and continue
7) Hard limits: max tool calls per turn, timeout, audit log

তোমার security gate already আছে (agent-tool assignment)। Controller যোগ করলে tool usage সত্যিকারের “persona-driven” হবে।

৮) WS JSON-এর বদলে binary pattern

বর্তমানে WS payload JSON string (StandardEvent) — debug-friendly, audit-friendly। Binary framing দরকার হয় যখন: (১) throughput খুব বেশি, (২) payload heavy, (৩) latency-critical streaming, (৪) mobile/low-bandwidth।

Recommendation (practical):

  • Keep JSON as canonical for correctness + observability.
  • Optional binary mode add করো via subprotocol/flag:
    • Client sends: {"type":"client.register","encoding":"bin-v1"}
    • Server switches to: ArrayBuffer frames
  • Binary format: CBOR / MessagePack safer than custom ad-hoc.
  • Audit log: server-side decode করে DB log অবশ্যই JSON/plain রাখবে।
Binary pitfalls (avoid)
- Custom bit-level schema without versioning → breaks clients
- No content-type/version field → cannot evolve
- Losing debuggability → harder incident response

If you go binary:
- versioned header (magic + version + eventId + payloadLen)
- keep a JSON mirror for ws_request_log
৯) Qdrant + BGE embeddings: directory indexing plan

তোমার চাহিদা: নতুন directory দেখা গেলে তোমার logic অনুযায়ী data index হবে। উদাহরণ path: CACHEDIR.TAG, models--BAAI--bge-large-en-v1.5, models--BAAI--bge-small-en-v1.5.

বর্তমান অবস্থা:

  • Repo-তে Qdrant integration code এখনো locate করা হয়নি (এই ডকুমেন্ট scope অনুযায়ী: যোগ করার প্রস্তাব/ডিজাইন দেয়া হলো)।
  • মডেল cache directories likely HuggingFace cache layout; CACHEDIR.TAG indicates cache semantics.

ডিজাইন: Offline-first vector indexer

Proposed components
A) File Watcher
- Watches configured roots (e.g. ~/.cache/huggingface, project data dirs)
- Ignores caches by default unless explicitly allowed
- Reads CACHEDIR.TAG to decide policies

B) Chunker
- For text: split by size (e.g. 800-1200 tokens) + overlap
- For code: split by file + function blocks

C) Embedder (BGE)
- Use one:
  - BAAI/bge-large-en-v1.5 (better quality, slower)
  - BAAI/bge-small-en-v1.5 (faster)
- Must run locally (no cloud) as per audit requirement

D) Vector DB (Qdrant)
- Collections: project_files, docs, logs
- Payload fields: path, mtime, sha256, chunk_id, language, repo, tags
- Index key: (path, sha256) to detect changes

Index policy for cache dirs:

  • Default: যদি directory-তে CACHEDIR.TAG থাকে → index skip (noise huge)।
  • Override allowlist: যদি তুমি models--BAAI--bge-* content metadata index করতে চাও (full weights না), তাহলে allowlist দিয়ে শুধু config/README/tokenizer files index করো।
  • Safety: binary weight files index না—শুধু metadata + naming + versions।
Qdrant query usage (expected)
- User asks: "এই ফাইলে টুল কন্ট্রোলার কোথায়?"
- Retriever returns top-k chunks
- Prompt assembly adds: [RAG_CONTEXT] minimal + citations (path:line)

This keeps prompt short while preserving intent.
